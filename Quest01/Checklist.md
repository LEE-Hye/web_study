> ## 1. HTML 표준의 역사는 어떻게 될까요?

<br>

* HTML이란❓

    - 웹 브라우저에 표시되도록 설계된 문서의 표준 마크업 언어   
        ###### 마크업 언어 : 태그 등을 이용해 문서나 데이터의 구조를 명시하기 위한 규칙을 정리한 언어

    - HTML은 버너스리가 인터넷에서 문서를 "HTML 태그"로 부르면서 시작


- 버너스리는 SGML 응용 프로그램이 되는 HTML을 고안해야 해 IETF에 의해 HTML 규격에 대한 최초의 제안을 간행물로 정의
     ###### SGML : 문서용 마크업 언어를 정의하기 위한 메타 언어


<br>

>> ### 1-1. HTML 표준을 지키는 것은 왜 중요할까요?

<br>

- 호환이 좋다.

- 의사소통이 쉽다.

- 평등하게 정보를 얻을 수 있다.

- 수정 및 유지 보수가 용이하다.

- 웹 접근이 쉽다.

<br><br>

>> ### 1-2. XHTML 2.0은 왜 세상에 나오지 못하게 되었을까요?

<br>

- XHTML 1.0이 후 몇 가지 제약사항을 추가해 제정을 진행하고 있었는데 차기 표준으로 HTML5가 제안 되었고 이미 여러 기능들이 공개되어 브라우저에서 HTML5 기능을 제공

- 웹 기반 사업과 기술을 표현하기에 XHTML 부족

<br><br>

>> ### 1-3. HTML5 표준은 어떤 과정을 통해 정해질까요?

<br>

- W3C와 WHATWG와 함께 HTML W/G를 결성해서 HTML5 표준 작성
    ###### W3C :웹 표준을 정의하는 국제 조직   
    ###### WHATWG : 애플, 구글, 마이크로소프트, 모질라 네 회사 중심으로 만들어진 웹 기술 표준화 조직

1. 초안(WD:Working Draft)
    - W3C 뿐 아니라 다른 곳에도 검토 받기 위한 문서

    <br>
2. 후보권고안(CR:Candidate Recommendation)
    - 광범위하게 검토를 받고 워킹그룹의 기술적 요구 사항을 만족 했다는 문서 공개
        ###### 워킹그룹 : 상위 조직에서 정한 주제나 목적에 따라 실제적으로 구체적인 일을 하는 모임

    <br>
3. 제안 권고안(PR:Proposed Recommendation)
    - 광범위한 기술 구현과 검토가 끝난 거의 완성된 문서로 최종 승인을 얻기 위해 자문위원회에 보냄

    <br>
4. 권고안(REC:Recommendation)
    - 모든 합의를 끝내고 W3C에 승인을 받은 문서

> ## 2. 브라우저의 역사는 어떻게 될까요?

<br>

* 브라우저란❓

    - 웹 페이지, 이미지 등의 콘텐츠를 수신, 전송 및 표현하는 소프트웨어

<br>

- 최초의 브라우저 월드 와이드 웹이 나오고 첫 그래픽 기반의 웹 브라우저인 모자이크가 나오고 그 이후로 점유율을 90% 찍은 넷스케이프, 인터넷 익스플로러, 모질라 파이어 폭스, 구글 크롬 등 많은 브라우저들 출시   

- 넷스케이프가 점유율 90%를 찍자 마이크로소프트는 익스플로러를 출시하며 거래처에게 윈도우를 주는 조건으로 넷스케이프를 쓰지 못하게 하고 마이크로소프트 윈도우 운영 체제에 익스플로러를 기본으로 설치하게 하는 등의 행동을 보였고 그 결과로 익스플로러를 쓰는 사람들이 더 많아져 IE는 점유율 90%를 찍었다. 또한 웹 표준에 맞지 않는 기능들을 구현하게 됐다.   
    => 1차 브라우저 전쟁


- IE가 브라우저 시장을 독점하니 개발에 소홀해지며 정체되고 보안의 문제까지 발생하게 되자 파이어 폭스와 크롬 등 웹 표준을 맞춘 브라우저가 등장하면서 IE보다 가볍고 빠르고 좋아 IE의 점유율은 파이어 폭스와 크롬보다 낮아졌다.   
    => 2차 브라우저 전쟁

- 지금은 모바일에서 인터넷을 사용하는 사람들이 많이 생겨 모바일에서 사용하기 편하게 업데이트를 하고 있기도 하다.

>> ### 2-1. Internet Explorer가 브라우저 시장을 독점하면서 어떤 문제가 일어났고, 이 문제는 어떻게 해결되었을까요?

> ## 3. HTML 문서는 어떤 구조로 이루어져 있나요?

<br>

```html
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>문서 제목</title>
    </head>
    <body>
        여기에 적힌 내용들이 웹 페이지에 나옴
    </body>
</html>
```

<br><br>

>> ### 3-1. `<head>`에 자주 들어가는 엘리먼트들은 어떤 것이 있고, 어떤 역할을 할까요?

<br>

- `<title>` : 브라우저 제목표시나 탭에 보이는 문서 제목
```html
<title>제목</title>
```

<br>

- `<base>` : 문서 안의 모든 상대 URL이 사용할 기준 URL 지정
```html
<base href="http://www.naver.com">
```

<br>

- `<link>` : 현재 문서와 외부 리소스 관계 명시
```html
<link href="css파일 위치" rel="stylesheet">
```

<br>

- `<style>` : 문서에 스타일 정보 입력
```html
<style>
    . {
        color : black;
    }
</style>
```

<br>

- `<meta>` : `<base>, <link>` 등으로 나타낼 수 없는 메타 데이터인 html 문서에 대한 정보 설정
```html
<meta charset = "UTF-8">
```

<br>

- `<script>` : 데이터와 실행 가능한 코드를 문서에 포함
```html
<script src="스크립트 파일 위치"></script>
<!--외부 스크립트 가져오기-->

<script>
    alert("Hello")l
</script>
<!--내부에서 스크립트 작성하기-->
```

<br>

- `<noscript>` : 스크립트 유형을 지원하지 않거나 브라우저가 스크립트를 비활성화 할 경우 보여줄 것 적기
```html
<noscript>
    <a href = "http://www.naver.com">네이버</a>
</noscript>
```

<br><br>

>> ### 3-2. 시맨틱 태그는 무엇일까요?

<br>

- 시맨틱 태그는 영역을 지정해주는 `<div>`와는 달리 `<header>, <footer>`같이 태그 자체에 의미가 있는 태그를 말한다.

<br><br>

>> ### 3-2-1. 시맨틱 엘리먼트를 사용하면 어떤 점이 좋을까요?

<br>

- 검색 엔진 최적화

- 쉬운 소스 코드 구조화

- 코드 가독성 향상

<br><br>

>> ### 3-2-2. `<section>`과 `<div>`, `<header>`, `<footer>`, `<article>` 엘리먼트의 차이점은 무엇인가요?

<br>

- `<section>` : 연관된 주제를 하나의 그룹으로 만듦
    - 관련 있는 내용을 묶음
    - 제목 태그를 함께 사용하는 경우 많음

    <br>

- `<div>` : 의미적으로 관계가 없을 때 사용

- `<article>` : 다른 영역에 영향을 주거나 받지 않고 독립적
    - 관련 있는 내용 중 독립적인 부분을 묶음
    - `<article>`안에 `<article>`이 들어가면 밖의 내용과 안의 내용이 관련이 있다는 뜻
    - 본문과 상관없이 태그 안의 내용만 봐도 하나의 콘텐츠일 때 사용

    <br>

- `<header>` : 맨 위에 웹 사이트 이름이나 내비게이션 묶음

- `<footer>` : 맨 저작권 정보, 작성자 등 적는 공간


<br><br>

>> ### 3-3. 블록 레벨 엘리먼트와 인라인 엘리먼트는 어떤 차이가 있을까요?

<br>

- 블록 레벨 엘리먼트
    - 요소를 사용했을 때 혼자 한 줄을 차지  
    ex) `<p> <ul> <div> <table>` 등

<br>   

- 인라인 레벨 엘리먼트
    - 요소를 사용했을 때 혼자 줄을 차지 않고 그 태그 부분만 차지  
    ex) `<a> <img> <span> <button>` 등

    - Markdown : 일반 텍스트 기반의 경량 마크업 언어
        - 일반 마크업 언어에 비해 문법이 쉽고 간단
        - HTML 등 서식 문서로 쉽게 변환 가능
